\# Trae CN SOLO 项目全局规则 # 项目名称：C# .NET8 医保结算微服务 # 技术栈强制锁定：.NET 8 ASP.NET Core WebApi、Nacos v2.3.x、MySQL8.0、EF Core、YARP网关、Polly、Serilog、Razor前端 alwaysApply: true language: 中文 mode: Solo Builder新项目初始化 / Solo Coder存量迭代  ## 一、架构硬性约束（最高优先级，必须严格执行） 1. 整体架构固定：多解决方案拆分    Gateway.Yarp（网关）+ ServiceCityA/ServiceCityB/ServiceCityC（3个地市独立微服务）+ Common.Core公共类库 + Web.Front Razor前端，固定5个项目，禁止随意新增/删减模块。 2. Nacos强制要求    1）所有业务服务必须自动注册Nacos服务发现，网关通过Nacos动态拉取服务实例，禁止硬编码IP、端口路由。    2）数据库连接串、超时时间、熔断阈值统一托管至Nacos配置中心，支持配置热加载。    3）Nacos存储必须对接MySQL，禁止使用内置Derby数据库。    4）命名空间固定dev，分组固定DEFAULT\_GROUP。 3. 数据库强制：MySQL8.0 + EF Core，严禁切换SqlServer、PostgreSQL；必须实现仓储模式，禁止原生拼接SQL。 4. 容错架构硬性规则    三个地市服务进程完全隔离，独立Polly熔断重试策略，单服务故障不得影响其余服务；网关自带健康检查，自动剔除Nacos下线实例。 5. 日志规范：统一Serilog，日志双输出：本地按日期日志文件 + MySQL异常日志表持久化。 6. 监控规则：内置BackgroundService后台定时心跳任务，服务心跳写入MySQL心跳表，前端实现红绿状态指示灯监控。  ## 二、项目目录与端口固定规则（禁止私自修改端口） 端口固定： 网关Gateway：5000 A市服务：5001 B市服务：5002 C市服务：5003 前端页面：5004 目录结构严格遵循标准.NET解决方案多层结构，Common.Core为所有项目共享引用，公共工具、实体、Nacos封装全部放在此类库。  ## 三、代码编码强制规范（C#标准） 1. 命名规范    类名、接口、方法：PascalCase大驼峰；变量、入参：camelCase小驼峰；私有字段下划线开头；常量全大写下划线分隔。 2. 格式：4空格缩进，大括号独行；单一职责原则，方法禁止超长，使用#region划分功能区域。 3. 注释：所有Controller、公共Service、核心工具类必须添加XML注释，复杂医保结算逻辑增加中文业务注释。 4. 安全强制    所有入参必须做参数校验；全局统一异常过滤器，捕获所有未处理异常，禁止直接抛出原始堆栈；禁止明文密码、密钥硬编码，敏感配置存放Nacos。 5. 接口标准：全部遵循RESTful WebApi规范，统一返回Result通用模型。  ## 四、Solo模式执行行为规则 1. Solo Builder（新建项目）：从零完整生成整套解决方案，自动配置全部NuGet依赖、Program主程序、配置文件、EF实体、数据库迁移脚本，生成可一键启动的完整架构，附带启动顺序说明。 2. Solo Coder（迭代修改）：执行修改前先确认文件白名单，非白名单核心配置文件（Nacos配置、网关配置、Db上下文），未经允许不得擅自改动。 3. 交付要求：每次开发完成，附带：启动顺序、环境前置软件清单、测试步骤；代码完成后必须校验能否正常注册Nacos、数据库可正常迁移。 4. 禁止行为    不得引入Redis、RabbitMQ等额外中间件（进阶优化除外）；不得改用Java、前端Vue/React等技术栈；不得简化微服务架构为单体项目。  ## 五、业务功能固定范围（仅开发指定模块） 1. 基础底座：Nacos注册配置封装、YARP网关动态路由、EF Core MySQL封装、Polly熔断重试、Serilog日志、后台心跳监控。 2. 业务核心：标准医保报文解析、入参校验、三地医保结算核算、模拟第三方医保接口调用、结算单据生成。 3. 前端演示：Razor简易页面，结算提交面板 + 三地服务在线状态监控面板（红绿指示灯）+ 日志查询页面。  ## 六、环境部署顺序规则（输出文档必须写明） 启动顺序固定：MySQL → Nacos服务端 → 3个地市业务服务 → YARP网关 → 前端Web项目。
